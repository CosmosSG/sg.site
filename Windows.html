Windows Setting

File Explorer

右クリックメニューの編集 (Windows11)

  • Windows10仕様に変更

    reg.exe add “HKCU\Software\Classes\CLSID\{86ca1aa0-34aa-4e8b-a509-50c905bae2a2}\InprocServer32” /f /ve

  • Windows11仕様に戻す

    reg.exe delete “HKCU\Software\Classes\CLSID\{86ca1aa0-34aa-4e8b-a509-50c905bae2a2}” /f

Command Prompt (cmd)

  • WindowsでUnixコマンドの pwd のようなことをする。

    @cdとすることで現在のディレクトリの絶対パスを得ることができる。

    C:\Users\UserName>@cd
C:\Users\UserName

Other

タスクバーにbatファイルをピン止めする

  1. batファイルのショートカットを作成

  2. 右クリック、プロパティから以下のように編集する

    cmd /c <Path>

時刻を自動で合わせる（PC内のボタン電池が切れてる時の対処）

Visual Studio Code (VSCode)

Install

インストール方法を見る

  1. インストール用のファイルをVisual Studio Code 公式サイトからダンロードする

  2. ファイルを実行する (VSCodeUserSetup-x64-1.80.2.exe)

  3. 使用許諾契約書に同意する

  4. インストール先の指定（特に理由がなければそのまま次に進む）

  5. スタートメニューフォルダーの指定（特に理由がなければそのまま次に進む）

  6. 追加タスクの選択（以下の二つの項目にチェックを追加し、次へ進む）

    ・エクスプローラーのファイルコンテキストメニューに [Code で開く] アクションを追加する
    ・エクスプローラーのディレクトリコンテキストメニューに [Code で開く] アクションを追加する

  7. 確認項目画面で確認できたら「インストール」を押す

    Error

    [ Markdown PDF ]

    PDFにExportしようとするとエラーが出る時がある。その場合以下の手順を行う

    1. Ctrl + Shift + p を押して以下のようなバーに打ち込む

      Preferences: Open User Settings (JSON)

    2. 開かれたSetting.json内の {　} の内側に以下のコードを入力する

      "markdown-pdf.executablePath": "ブラウザPath"

  • Markdown Preview Enhanced または Markdown Preview Github Styling

C言語

環境構築

環境構築方法を見る

  1. インストール用のファイルをCygwin公式サイトからダウンロードする ( setup-x86_64.exe )

  2. ダウンロードを押すと以下のような警告が出る時がある。

    警告が出たときはこちら

    1. [ ･･･ ] から [ 保存 ] を押す

    2. 以下の画面の [ 詳細表示 ] を押す

    3. [ 保持する ] を押す

    4. ダウンロードフォルダから setup-x86_64.exe を見つけ、右クリックをして [ プロパティ ] を押す

    5. 下のセキュリティの項目の [ 許可する ] にチェックを入れて [ OK ] を押す

  3. setup-x86_64.exe を実行し、[ 次へ ] を押す

  4. インストール手段の選択（特に理由がなければそのまま [ 次へ ] を押す）

  5. インストール場所の選択（特に理由がなければそのまま [ 次へ ] を押す）

  6. パッケージの置き場所の選択（適当なディレクトリを選択し、[ 次へ ] を押す）

  7. 接続方法の選択（特に理由がなければそのまま [ 次へ ] を押す）

  8. ダウンロードサイトの選択を行う（選択したら [ 次へ ] を押す）

  9. Viewを Full にして、Searchに gcc-core と入力する

  10. gcc-coreを見つけたら、Skipとなっている項目をクリックし、最新バージョンを選択する
    ※ この時点では 11.4.0-1 が最新なのでこれを選択する

  11. 確認画面が出てくるので [ 次へ ] を押す

  12. 特に理由がなければそのまま [ 完了 ] を押す

  13. Cygwin64 terminal を立ち上げ、gcc --version を実行する

    $ gcc --version
gcc (GCC) 11.4.0
Copyright (C) 2021 Free Software Foundation, Inc.
This is free software; see the source for copying conditions.  There is NO
warranty; not even for MERCHANTABILITY or FITNESS FOR A PARTICULAR PURPOSE.

    このようにバージョンが表示されれば、環境設定完了

デフォルトでは C:\cygwin64\home をホームディレクトリとして設定されている。
これを、C:\Users にするには以下の手順を行う

ホームディレクトリの変更方法を見る

  1. エクスプローラーを開き、Cドライブを開く

  2. 上の手順でインストールした場合はここにcygwin64というディレクトリが生成されるため、cygwin64のディレクトリを開く

  3. home -> usrnameの順に開き、中にあるファイルを C:\Users\username に移動させる

    詳しく見る

    1. 移動手順1: ファイルを選択

    2. 移動手順2: 上のリボンから [ 移動 ] を押し、[ 場所の選択 ] を押す

    3. 移動手順3: 下の フォルダー(F): の値が自分のユーザ名になっていることを確認し、[ 移動 ] を押す

  4. 空になった C:\cygwin64\home を削除する

  5. Win + R でファイル名を指定して実行を開き、cmdと入力した後、Ctrl + Shift を押しながら Enter を押し、管理者権限でコマンドプロンプトを開く

  6. 管理者権限でコマンドプロンプトを開けたら、以下のコードを入力し、実行する

    mklink /D C:\cygwin64\home C:\Users

  7. このように出力されたら完了

    C:\cygwin64\home <<===>> C:\Users のシンボリック リンクが作成されました

  8. Cygwin64 terminalを開き、以下を入力すると C:\Users\username が開かれていることがわかる

    ls

  9. ルートディレクトリから開きたいときはpathの先頭に /cygdrive/ と入れないといけないことに注意
    以下はルートディレクトリからDocumentsを開いた例

    cd /cygdrive/c/Users/username/Documents/
